# Lessons Learned Applied from the V4.2 Publication Effort

The first V4.2 GitHub publication effort taught that a technically coherent repository can still be misunderstood if the public front door does not explain what kind of work the reader is seeing.

This 5G package therefore applies the following publication gates.

## 1. Start with the model category

Toy Model 5G is not presented as a completed physical theory. It is presented as a reproducible falsification and metric-reconstruction protocol inside the V4.2 integrated geometric research architecture.

## 2. Explain the purpose before the files

The purpose is to test whether galaxy rotation-curve residuals are better treated as a relational metric/connection reconstruction problem than as a direct missing-acceleration patch.

## 3. Preserve the continuity line

V4 introduced projection intuition. V4.1 introduced the Embedded Observer and Relationship -> Organization -> Observable Order frame. V4.2 added toy-model structural compatibility. 5G is the first expanded real-data diagnostic track.

## 4. Do not bury the guardrails

The public baseline repeats the non-claims clearly. It does not claim dark matter has been solved, physical 3T has been proven, or Lambda-CDM has been replaced.

## 5. Show failures as part of the method

5G A-to-K is included as a failure trail. It documents scalar threshold failures, kernel limitations, scheduling-link failure, Laplacian relaxation failure, and the shift toward metric reconstruction.

## 6. Explain the next mathematical object

The next step is 5G-N: a relational metric generation law. 5G-L/M reconstructed the shadow of the engine; 5G-N must attempt to write the engine.
