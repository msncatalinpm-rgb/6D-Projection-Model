# Upload Strategy — Version 4.1

## Principle

Upload a clean, disciplined package.

The first V4.1 upload should include:

- root `README.md`
- `LICENSE`
- the combined PDF review package in `paper/`
- individual Markdown documents in `documents/`
- individual PDF documents in `documents/pdf/`

Avoid adding unfinished notes, speculative side discussions, or raw conversation transcripts in the first release.

## Upload Order

1. Create or update the repository.
2. Upload `README.md`.
3. Upload `LICENSE`.
4. Create `paper/` and upload the combined PDF.
5. Create `documents/` and upload the Markdown documents.
6. Create `documents/pdf/` and upload the individual PDFs.
7. Commit using the approved commit message.
